## Version Numbering Scheme

```none
YYYY.RELEASE.FIX
```

- YYYY represents the year
- RELEASE indicates the number of releases made in that year
- FIX starts at 0 and increments when bug fixes are applied
  - Example versions:
    - 2021.1.0
    - 2021.3.1

### Canary Versions

```none
YYYY.RELEASE.FIX-canary.CANARY-RELEASE
```

- We may periodically release development versions as canaries before official releases
- CANARY-RELEASE starts at 0 and increments sequentially (1, 2, 3, ...) until release

- Example versions:
  - 2021.1.0-canary.0
  - 2021.3.1-canary.3

## Questions and Inquiries

**Please read the documentation and test the software first before asking questions**

Please submit your questions to the appropriate channels on Discord.
If you're unsure which Discord channel to use,
please check the `#help` channel to determine where to post your question.

Shiguredo members will prioritize responding to questions from users who hold the **Contribution Label**.

For programming language or framework usage questions, please refer to [Stack Overflow](https://stackoverflow.com/) or other relevant resources.

### Troubleshooting Issues

Please specify what information is needed to resolve the problem. We'll need detailed reproduction steps to attempt a fix.

### Usage Questions

Please explain what you're trying to achieve.
Please provide clear and detailed information using tools like Gist to help us understand your situation.

### Customization Questions

**Shiguredo members and administrators do not respond to customization-related questions**

- For customization inquiries, we will respond with: "Shiguredo members and administrators do not respond to customization-related questions"
- Please note that it's perfectly acceptable to ask questions on Discord

### Build-Related Questions

**Shiguredo members and administrators do not respond to build-related questions**

- Please use pre-built binaries
- Those wishing to build from source should refer to the GitHub Actions configuration
- For build-related questions, we will respond with: "Shiguredo members and administrators do not respond to build-related questions"
- Please note that it's perfectly acceptable to ask questions on Discord

## Bug Reporting

Please first submit bug reports to the appropriate channels on Discord.
Please note that we cannot guarantee response times or resolution, regardless of whether the user has purchased the Sora license.

## GitHub Issues

Please only use GitHub Issues when requested by Shiguredo members or administrators after consulting on Discord.

## GitHub Pull Requests

For submitting improvements or bug fixes via Pull Request, please first consult with Shiguredo members or administrators via Discord.

Once improvements or fixes are merged, they require ongoing maintenance. We therefore require "discussion and persuasion" from the contributor.
If this proves difficult, please consider forking the project. Shiguredo's OSS development processes are closed.
